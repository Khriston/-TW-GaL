1. Abstract

GameIT este o aplicație Web pentru a usura invatarea unuia dintre jocurile: Table, Dame și X și O. Acest document descrie arhitectura aplicației, funcționalitățile aplicației și tehnologiile folosite în cadrul procesului de dezvoltare.

2. Introducere

Se pot găsi nenumărate turotiale pentru aceste jocuri pe internet, însă aplicația noatră web dispune și de o scurtă istorie a jocului, indicații pentru începători și un chestionar, pe dificultăți diferite, pentru testarea cunoștințelor.

Obiectivele noastre sunt:

  • Furnizarea unei modalități rapide și ușor de utilizat de învătare pentru jocurile menționate mai sus.
  • Oferirea unei modalități de testare a cunoștintelor.
  • Compatibilitatea deplină a aplicației cu Web browsere moderne.

Mai multe detalii se găsesc în lista de proiecte.

3. Învătarea

Pentru a clarifica funcționalitățile aplicației, am enumerat cerințele utilizatorilor, ordonându-le după importanță.

4. Tehnologii folosite

În cadrul dezvlotării proiectului vom folosi tehnologii moderne, ubicuue la nivelul industriei. Ca server de aplicații Web vom folosi XAMPP deoarece este cross-platform, oferindu-ne o gamă largă de sisteme de operare pe care putem lucra.

4.1 Front-end

Deoarece componenta de front-end a proiectului (interfața) este simplă vom folosi doar ultimele versiuni ale standardelor HTML și CSS și limbaj de scriptare precum JavaScript. Pentru structurarea paginilor vom folosi HTML5 și pentru stilizarea paginilor vom folosi CSS3.

4.2 Back-end

Pentru dezvoltarea back-endului vom folosi:

  • MySQL pentru baza de date deoarece este open-source și des utilizat.
  • PHP 7 ca limbaj de scriptare deoarece:
    • are suport pentru Unicode (absolut necesar pentru o aplicație Web modernă)
    • este mai performant decât versiunile anterioare
    • are suport pentru variabile tari tipizate (folositor la depanare)

5. Detalii de implementare

Există mai multe metode de implementare a funcționalităților descrise (Învătarea). Pentru a clarifica structura și modul de funcționare al aplicației vom prezenta arhitectura componentelor principale mai detaliat.

5.1 Arhitectură

Din punct de vedere arhitectural, aplicația este monolitică. Diferitele funcționalități precum interfața grafică, logica de lucru și accesul la date nu sunt separate pe diferite "straturi" ci sunt puternic cuplate. Aproape fiecare pagină este responsabilă pentru propria interfață grafică, logică și acces la date, singura excepție fiinđ folosirea unor pagini "template", pe care le încarcăm în fiecare pagină, pentru lizibilitate și ușurința mucii.
